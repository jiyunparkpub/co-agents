## Critical 이슈

### C1. THE FACT 상한의 근거 불일치 — 단위 혼용
- 위치: prd.md §FR-A1, §3.1, §11 오픈이슈 1
- 문제: FR-A1은 참조 가이드의 "소스당 [기준] 이하"를 인용하면서도 서로 다른 단위로 귀결시킨다. 가이드는 (a) 소스당 기준, (b) 양적 비율의 두 가지 근거를 제시하나 본 PRD는 이를 구분하지 않는다. **"종합 결과물 전체 총량"에 소스당 기준을 적용할 것인지, 소스별 기여도로 쪼개어 제한할 것인지**가 불명확. FR-A1 자체가 "측정 방법 자체를 뒤로 미뤘으므로 핵심 지표 달성 가능성이 불확정".
- 근거: 참조 가이드의 실무적 지침 — **"소스당"**이라는 단위가 명시적.
- 수정 방향:
  1. 소스당 상한과 종합 결과물 전체 상한을 **두 개의 별도 하드 규칙**으로 분리 정의.
  2. 소스별 기여도 측정은 기존 유사도 엔진 재사용 — `[TECH CHECK]` 제거하고 Must-have 구현 범위에 포함.
  3. 수치를 유지한다면 "왜 해당 값인지"를 양적 기준 또는 환산 근거로 별도 문단으로 기술.

### C2. 상표권·환각 귀속 리스크 완전 누락
- 위치: prd.md §2.1 리스크표, §FR-A5, §8 리스크 맵
- 문제: 참조 판례는 "비축자적 요약 침해" 외에 **"AI 환각으로 브랜드 명의 허위 콘텐츠 생성이 상표권 책임 창출"**이라는 별도 쟁점을 확립. [PRODUCT]는 카드 상단에 매체명을 부착해 표출하며, AI 환각 시 해당 매체가 보도하지 않은 내용이 "매체명 + 본문" 조합으로 고정 노출 → **동일 리스크 구조**. 현 PRD는 기존 플래그만 언급하고 "매체-내용 귀속 왜곡"의 검출·차단 프로세스 부재.
- 근거: 참조 가이드 — "AI 환각에 대한 상표권 책임도 창출". FR-B3는 매체명 필수 표시 의무화 → 결과적으로 "매체가 보도한 사실"로 독자가 인식.
- 수정 방향:
  1. FR-A5 유사도 검사 외에 **"매체-팩트 귀속 검증"** 별도 Must-have 추가.
  2. 검증 실패 시 플래그 도입 → 에디터 검수 큐 강제 이동.
  3. §8 리스크 맵에 "상표권·허위 귀속 리스크" 별도 행 추가 + §9.1 보험 범위에 상표권 커버리지 명시.

## Major 이슈

### M1. 성공 지표 베이스라인·측정 경로 부재 → 측정 불가
- 위치: prd.md §3.1
- 문제: (a) 핵심 CTR "현재 미측정 → [목표치]"의 베이스라인 수집 계획이 W1~W8 어디에도 없음. (b) "유사도 위반률 < [기준]"의 분모와 분자 정의 미기술. (c) "감지기 Precision/Recall [기준]"은 테스트셋 구성 방법·규모 부재.
- 근거: PRD 공통 규칙 — Primary/Secondary/Guardrail 모두 **측정 가능(measurable) + 기한** 필요.
- 수정 방향: ① W1에 CTR 측정 인프라 및 베이스라인 수집 마일스톤 추가, ② 유사도 위반률 분모·분자를 구체적으로 명시, ③ 감지기 평가셋 규모 지정.

### M2. Tier 분기 기준의 소스 매핑 부재 + 기존 컬럼과 충돌
- 위치: prd.md §FR-B1, §FR-A6, §11 오픈이슈 7
- 문제: Tier 1/2/3 분기가 PRD 핵심인데, **기존 소스들을 어떤 Tier로 초기 분류하는지** 표가 없다. 기존 스키마의 `tier` 컬럼은 **신뢰도·편향 축**이지 라이선스 축이 아니다. FR-A6가 추가하는 `license_code` 컬럼과 기존 `tier`의 병존·충돌 규칙 미정. 오픈이슈 7을 에디터 판단으로 미뤘는데, Tier 비중 목표(§3.2)의 분모를 좌우하는 **구조적 설계 결정**.
- 근거: 기존 스키마 주석에 신뢰도 레이블임이 명기. 참조 가이드 Tier 정의는 라이선스 축.
- 수정 방향: ① `sources`에 `license_tier` 컬럼을 기존 `tier`와 **독립** 추가, ② 소스를 라이선스 Tier로 초기 분류한 부록표를 PRD에 첨부, ③ 기본값 `unlicensed`(안전 우선) 명시.

### M3. 번역 금지 원칙이 독립 FR로 승격되지 않음
- 위치: prd.md §FR-A1·A3 하위에 묻힘
- 문제: 브리프·가이드의 핵심은 "원문 직접 번역 금지, 팩트만 추출하여 **독자적 한국어**로 새로 작성". PRD FR-A에는 **독립 FR로 승격되지 않고** 다른 항목의 하위처럼 다뤄진다. **원문-출력 구조 거리**를 어떻게 후처리로 검증할지도 누락.
- 근거: 참조 가이드 — "번역은 2차적 저작물, 팩트 기반 독자적 서술은 아니다". 참조 판례 — "축자적이지 않더라도 구조·순서·톤·표현 선택 반영 시 침해".
- 수정 방향: FR-A에 "A0. 팩트 추출 후 독자적 한국어 재서술(번역 금지)"를 최상위 원칙으로 승격. FR-A5에 "문장 정렬·구조 거리 검증" 추가.

### M4. 관련 법규 상 법적 지위·책임 분배 미검토
- 위치: prd.md §FR-B6, §FR-C2, §6 NFR 관할법
- 문제: FR-B6은 고지를 "법적 책임 분배 불명"으로 인식하고 투명성 배너를 제안하나, 관련법상 **서비스 유형 간** [PRODUCT] 위치 판단이 없다. 편집·변환을 적극 수행 → **규제 강도 높은 쪽으로 분류될 위험**이 있고, 이는 등록·정정보도 의무 범위 전면 확대로 이어짐.
- 근거: 참조 가이드 — "뉴스 애그리게이션 플랫폼은 이 범주로 분류될 수 있으며, 정정보도 청구 표시 의무와 매개된 콘텐츠에 대한 잠재적 책임을 부담한다".
- 수정 방향: ① §6에 "법적 지위 결정: 법률 자문 Must-have" 추가, ② FR-B6 고지 카피 스펙 상세화, ③ FR-C2 정정보도 SLA를 관련법 조항과 정렬.

### M5. CTR 목표의 근거 부실 + 내부 지표 트레이드오프 공백
- 위치: prd.md §3.1, §3.3 Guardrail
- 문제: 유일 근거가 [경쟁 서비스] 공개 수치 + [가정]. 한쪽은 **B2B 유료 모니터링** 사용자 행동으로 일반 독자 소비 CTR과 동일 축이 아니다. 또 하나는 검증되지 않은 가정. 한편 UX는 **[PRODUCT] 내부 체류·공유 수익화를 구조적으로 약화** — 이 트레이드오프가 Guardrail에 없음(체류시간·재방문률 가드레일 부재).
- 근거: PRD 공통규칙 — "지표 간 충돌 가능성 검토" 필요.
- 수정 방향: ① [목표치]를 "내부 A/B 베이스라인 + 유사 서비스 공개 데이터" 기반 근거로 교체하거나, 상대 목표로 전환, ② Guardrail에 체류시간 / 재방문률 추가.

## Minor 이슈

### m1. 판례 진행 단계 서술 과장
- 위치: prd.md §2.2
- 문제: "약식판결 브리핑 마감 … 변환성 추정이 무너짐"은 **브리핑 마감 ≠ 판결**이므로 단정 과장.
- 수정 방향: "변환성 추정이 무너짐" → "변환성 추정에 의문이 제기됨"으로 완화.

### m2. 신규 포트 네이밍 컨벤션 불일치
- 위치: prd.md §9.2
- 문제: 기존 Port/Repository 의미 구분과 일관되나, 일부 수식이 중복. 짧은 형태가 기존 축약 컨벤션에 부합.
- 수정 방향: 네이밍 컨벤션 한 줄 명시 후 정렬.

### m3. 옵트아웃 포털 본인 확인 절차 미정
- 위치: prd.md §FR-C1, §4.3
- 문제: 최소 정보만으로 삭제 수행하면 **제3자 허위 신고 → 합법 콘텐츠 삭제** 리스크. 관련법상 허위 통지 책임은 실제로는 사후 구제라 사전 예방 필요.
- 수정 방향: FR-C1에 "도메인 이메일 검증 + e-서명 진술서" 추가.

### m4. Tier 분기의 SSR 쿼리·캐시 영향 미기술
- 위치: prd.md §6 NFR
- 문제: 컴포넌트 variant SSR은 신규 컬럼 조인이 필요 → 기존 캐시 직렬화 포맷 변경. 성능 영향 미평가.
- 수정 방향: W1 산출물에 쿼리 수정 범위·캐시 키 마이그레이션 명시.

## 강점 (다음 라운드에서도 유지)

- **판례 인용의 정확성**: 참조 가이드 원문과 사건명·쟁점 일치. 비구속성도 정확히 짚음.
- **Tier 기반 UI 분기(FR-B1)**: 참조 가이드의 리스크 최소화 아키텍처와 일대일 매핑. 단일소스·opinion·CC-ND 일괄 처리 구조 우수.
- **컴플라이언스 인프라 Day-1 도입(FR-C)**: 지정 대리인·옵트아웃·정정보도·감사 로그를 한 트랙으로 묶어 참조 가이드의 체크리스트와 전량 정합.
- **차별화 요소 전면화(FR-B7)를 "법적 방어 = 제품 차별화"로 통합**: 방어 비용을 차별화 레버로 재정의한 프레이밍이 전체에 일관되게 녹아 있다.
- **[마일스톤 N]와 FR 번호가 전 구간에 매핑**되어 하류가 단위별로 착수 가능.

3. **M1**: §3.1 지표별로 분모·분자·측정 인프라·베이스라인 수집 시점을 표로 확장.

5. **M3**: "번역 금지 + 독자적 한국어 재서술"을 FR-A0(최상위 원칙)으로 승격. 문장 정렬·구조 거리 검증 방법 구체화.

7. **M5**: CTR 목표 근거 보강 또는 상대 목표로 전환. Guardrail에 체류시간·재방문률 가드레일 추가.
8. **Minor m1~m4** 반영.
