## 잔존 이슈

### Critical (0건, 목표 달성)
(없음)

### Major (1건, 목표 ≤ 2 달성)

**M6(신규·M5 부분반영 파생). CTR 베이스라인 수집 경로의 프록시 모호성**
- 위치: §3.1 표 첫 행 "베이스라인 수집" 셀, §10 W0 행
- 문제: W0에 "프록시 배포로 수집"이라 기술했는데, 수집 환경이 **Tier 분기 + UI 개편 이후의 노출 환경과 동일하지 않다** — 즉 상대 목표의 배수는 UI 개편 효과와 콘텐츠 효과가 섞여 **공정이용 방어 증거로 제출 시 인과 분리 불가**. 또 절대치 하한의 [경쟁 서비스] 공개치는 출처 미명시.
- 등급: Major(법적 방어 증거 능력 약화, 단 측정 자체는 가능 → 블로커 아님)
- 수정 방향(Phase 2 Design에서 반영 가능): ① W0 수집 구간 중 일부는 현 UI, 나머지는 interim UI를 A/B로 병행해 UI 효과와 콘텐츠 효과를 분리, ② 하한 근거의 [경쟁 서비스] 출처·수집일자 명시 또는 삭제.

### Minor (3건 잔존 + 파생)

**m5. 고유 콘텐츠 비율 [기준] 측정 정의 미기술**
- 분자 vs 분모 계산 시 포함/제외 여부 모호. Phase 2 Design에서 컴포넌트 영역 구분 확정 후 집계 쿼리 스펙으로 재명시 필요.

**m6. FR-A5.5 팩트 원자 분해의 파서 선택 기준 부재**
- 비용·지연·정확도 트레이드오프 축이 다름. §11 오픈이슈와 병합해 W1 벤치 범위에 명시 포함 권장.

**m7. §8 리스크 맵 "검사 비용 [가정]"의 계산 근거 불명**
- 아이템 단위 vs 팩트 원자 단위 호출 → 실제 호출 수는 수배 가능. W1 실측 대비 필요.

## 신규 이슈 (v2 수정 중 발생)

### N1. FR-A1 (b) 환산 계수 근거가 [가정] 하나로만 지탱
- 위치: §FR-A1 (b)
- 문제: 계수의 통계·연구 인용 없음. §11 오픈이슈로 이관되어 "W1 실측 재조정"이 약속됨 → **블로커 아님**. 단 실측 결과에 따라 양적 기준과의 정렬이 약해질 수 있음.
- 권장: Phase 2 핸드오프 시 "W1 실측 후에도 클러스터 전체 원문 길이의 [기준]를 초과하지 않는 상한 락" 한 줄 추가.
- 등급: Minor

### N2. FR-B7 문구가 Phase 2 범위 축소 여지 남김
- 핵심 차별화 축인데 "Design 단계에서 확정"이 모호하게 남아 있어 후순위 밀림 위험.
- 권장: 핸드오프 메시지에서 위치 하드 고정 명시.
- 등급: Minor

### N3. FR-C3 등록 vs 페이지 오픈 시점의 정합성
- 페이지가 오픈되는데 지정 대리인 등록은 [에디터 확인 필요]로 남음 → 페이지 오픈 vs 세이프하버 발효 사이 공백.
- 권장: §10에 "릴리스 전 등록 완료 선행" 주석 추가.
- 등급: Minor

## Phase 2 핸드오프 메시지 (Design Blue Team에게)

1. **Tier 분기는 법적 판정의 1차 가드** — `sources.license_tier` × `news_items.content_type` 매트릭스에서 파생 조건을 컴포넌트 variant에 반드시 인코딩. 이름 충돌 방지를 위해 디자인 네이밍은 `LicenseVariant` 권장.

2. **Outlink CTA = 프라이머리**, 헤드라인과 동등 위계 + 모바일 sticky. **핵심 차별화 섹션은 직후 위치로 하드 고정**. 매체 로고·이름과 차별화 섹션은 **시각적으로 분리**(상표권 원칙). 카드 내에 "AI 종합(매체 원문 아님)" 구분 라벨 영역 확보.

3. **LicenseBadge + SourceMeta + AI 고지 배너 + "AI 종합" 라벨** 4요소가 같은 row·인접 row에서 시각 충돌하지 않도록 토큰·간격 재설계. WCAG 2.2 AA, `aria-label`·툴팁·키보드 포커스 모두 스펙에 포함. 캐시 네임스페이스 전환에 맞춘 서버 컴포넌트 경계도 함께 검토.

4. **ShareSheet 기본값 = 원문 URL**. OG 카드 이미지에 본문 금지. 보조 옵션은 차별화 요소 1줄만. `src/.../components/<module>`는 전면 재작성 대상.

5. **W0 A/B 플랜(잔존 M6 대응)**: W0 베이스라인을 복수 셀로 병행 수집하여 UI 효과와 콘텐츠 효과를 분리. Phase 2 Design 산출물에 interim UI 시안을 반드시 포함.

6. **컴플라이언스 포털 3종**은 ko/en 동일 IA로. 본인 확인 UX를 폼 스텝으로 시각화. 관련법 SLA 타이머를 사용자에게 가시화.
