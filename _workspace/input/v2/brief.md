# Brief — RSS 법적 리스크 대응: 데이터 파이프라인 & UX 재설계 ([PRODUCT] v2)

> **📝 샘플 문서** — 이 파일은 Multi-agent Producer-Reviewer 파이프라인의 **입력(brief) 예시**로 공개된 샘플입니다. 원본의 사업 구체 내용은 플레이스홀더로 일반화되어 있으며, 본인의 프로젝트 맥락에 맞게 각 섹션을 다시 작성해 사용하시면 됩니다.

## 배경

`agents/reference/rss-legal-guide.md`(2026-04 기준)가 RSS 기반 AI 뉴스 애그리게이션의 법적 리스크 지형을 정리했다. 핵심:

1. **RSS는 상업적 재사용 라이선스가 아니다** — _MidlevelU v. ACI_ (2021) 판결
2. **AI 요약 자체가 안전하지 않다** — _Advance Local v. Cohere_ (2025): 원문의 구조·순서·톤·표현을 반영하는 비축자적 요약도 침해 가능
3. **한국 정부 2026-02 가이드라인**: AI 뉴스 요약/브리핑을 "공정이용 불인정 가능성 높음" 사례로 명시
4. **번역 금지 원칙**: 원문 번역은 2차적 저작물(베른 협약). 팩트 추출 + 독자적 한국어 재서술만 가능
5. **아웃링크 모델이 최저 리스크**: _Ground News_ 스타일(메타데이터 집계 + 원문 클릭 유도)

[PRODUCT]는 이 가이드를 제품 설계의 중심에 놓아야 한다 — **"원문을 대체하지 않고, 원문에 없는 가치를 추가"**.

## 변경 범위

### A. 데이터 수집·가공 (`src/.../<module>`)

현재 프롬프트 상태:
- `clario.ts` — 환각 방지 + 저작권 방어 규칙(리드 추적 금지, 단일소스 5단어 유사도 상한, 복수소스 종합 의무) **부분 반영**
- `koreaAngle.ts` — Korea's Angle을 "가장 변환적인 콘텐츠"로 포지셔닝 + 재프레이밍 예시 **반영**
- `scoring.ts` — 클러스터링 강화, 라이선싱 상태(`licensed`/`cc`/`unlicensed`) 입력, `license_risk` 출력 **반영**

**추가 검토 필요**:
- THE FACT 200자 하드 상한의 근거 재점검 (Nihon v. Comline "소스당 50단어 이하" 권고 대비)
- 리드(첫 문단) 자동 감지/차단 로직 (현재 프롬프트 규칙만 존재)
- 복수소스 종합 강제 (relatedArticles 없으면 스킵 옵션 or unlicensed 건 자동 보류)
- 의견/사설/기획기사 자동 분류 → 별도 안전 정책 (현재 구분 없음)
- AI 요약 유사도 자동 검사 (post-processing)
- CC-ND 콘텐츠의 번역 금지 (The Conversation 등)

### B. UI/UX 법적 리스크 최소화

현재 UI 분석 필요 지점:
- NewsCard에 THE FACT/CONTEXT/FOR KOREA 전체 표시 → **원문 대체 리스크**
- 공유 OG 카드에 요약 일부 포함 → **재배포 리스크**
- 원문 링크가 "원문 보기" 텍스트로 약하게 처리 → **클릭률 낮을 시 공정이용 불리**
- 매체명·발행일·라이선싱 상태 표시 부족 → **출처 표시 의무**
- 에디터 검수 상태("AI 요약 · 에디터 검수")는 있으나 법적 책임 분배 불명
- 정정보도·삭제 요청 포털 부재 → **언론중재법·DMCA 의무**

**UX 목표**:
1. **원문 유도 강화**: 아웃링크를 UI의 1차 CTA로 승격 (Ground News 레퍼런스)
2. **Tier 기반 표시 분기**: 라이선싱/CC/비라이선싱에 따라 표시 수준 차등 (예: 비라이선싱은 헤드라인+50단어 이하 팩트만)
3. **투명성 강화**: 매체명·발행일·편향·라이선스 상태·AI 관여도를 명확히 표기
4. **[PRODUCT] 고유 가치 전면화**: WHY IT MATTERS / FOR KOREA / Korea's Angle / 다관점 비교를 시각적으로 우선 배치(요약보다 전면에)
5. **컴플라이언스 인프라**: 정정보도 청구·삭제 요청·옵트아웃 포털, 저작권 고지 페이지, robots.txt 준수 기록

## 결과물 요구사항

### Phase 1 (PRD) 포함 항목

- 문제 정의: 법적 리스크 스펙트럼(HIGH/MEDIUM/LOW) + 2026년 환경 변화
- 목표/성공 지표: 원문 클릭률 CTR, 유사도 탐지 F1, 옵트아웃 응답 SLA
- 사용자 시나리오: (a) 브리핑 리딩 → 원문 이동, (b) 공유, (c) 정정/삭제 요청
- 기능 요구사항: 데이터 파이프라인 변경 + UI 변경 + 컴플라이언스 인프라
- 범위 제외: 라이선스 계약 비즈니스(별도 트랙), EU 제15조 대응(해외 확장 시)
- 리스크 맵: 변경사항 반영 후 잔존 리스크

### Phase 2 (Design v2) 포함 항목

- IA 재구성 — 원문 링크 승격 반영
- 컴포넌트 스펙: NewsCard(Tier별), OutlinkCTA, LicenseBadge, SimilarityWarning, [CorrectionFormComponent]
- 공유 플로우 재설계 — 요약 최소화, OG 카드 메타데이터 기반
- 설정/푸터: 옵트아웃, 정정보도, 저작권 고지
- 모바일 레이아웃 변경
